## 2024-05-20 - [Centralized String Utilities Performance]
**Learning:** `json_escape` implementation heavily relies on memory allocations when using `std::ostringstream`. By switching to `std::string::reserve` combined with manual character appending, performance improved drastically. Additionally, returning `std::string_view` from `trim()` prevents unnecessary heap allocations across the extensive config and telemetry parsers. However, care must be taken to not create dangling string views from temporary `.substr()` responses.
**Action:** Always prefer `std::string::reserve` over `std::ostringstream` for predictable JSON escaping, and use `std::string_view` for high-frequency string utilities while ensuring explicit `std::string` casting when interfacing with older C++ APIs like `std::stod` or `std::map`.
