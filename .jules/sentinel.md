## 2024-05-15 - [API Key Exposure in Process Listing]
**Vulnerability:** The API key for language model access was being passed via command-line arguments to `curl` (`-H Authorization: Bearer <key>`) in `src/model/gateway.cpp`. This exposes the plaintext key in the system's process listing (`ps aux`) to any user on the machine.
**Learning:** Command-line arguments of spawned processes are globally readable on most POSIX systems. Sensitive data, especially secrets and API keys, should never be passed as plain text in `argv`. For `curl`, a secure alternative is passing headers via a file using the `-H @<filepath>` syntax.
**Prevention:** Use a secure temporary file creation mechanism (like `mkstemp()`) with proper error handling to write secrets to disk (ensuring restricted permissions). Then, configure the external command to read the secret from the file instead of the command line. Ensure the temporary file is deleted immediately after the external process completes, even on error paths.
